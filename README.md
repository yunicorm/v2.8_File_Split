# Path of Exile マクロ v2.8.1

## 概要

このマクロは、Path of Exileにおける特定のビルド（Wine of the Prophet使用）向けに最適化された自動化ツールです。マナ管理、スキル自動実行、フラスコ管理などを統合的に制御します。

## 主要機能と動作原理

### 1. Wine of the Prophet システム

**概要**：Enkindling Orbによって強化されたMana Flaskの効果を最大限活用するシステム

**動作原理**：
- **Enkindling Orb効果**：効果時間中（6.05秒）チャージ獲得不可、77%効果増加
- **チャージ獲得式**：`1×(1.50+ManaB%)×1.77 チャージ/秒`
- **The Tides of Time**：フラスコチャージ獲得 +50%
- **Mana Burn**：最大340%の追加効果

**タイミング調整**：
```
経過時間     | Wine使用間隔
0-60秒      | 22-22.5秒
60-90秒     | 19.5-20秒
90-120秒    | 17.5-18秒
120-170秒   | 16-16.5秒
170秒以上   | 14.5-15秒
```

### 2. マナ監視システム（完全枯渇検出式）

**検出方式**：
- 円形マナオーブの下部3つの高さ（85%、90%、95%）で検出
- 各高さで5ポイント、合計15ポイントをチェック
- 全ポイントで青色が検出されない場合、完全枯渇（0/26）と判定

**座標設定**：
- マナオーブ中心：X=3294, Y=1300
- 半径：139ピクセル
- 青色閾値：40以上（青が赤・緑より20以上大きい）

**重要な設定**：
- ゲーム内で「Always Show Mana Cost」を**必ずOFF**にする
- リザーブされたマナを非表示にする

### 3. Tincture管理システム

**基本動作**：
1. マクロ開始時：即座にTincture（3キー）を使用
2. マナ完全枯渇検出時：Tinctureが切れたと判定
3. 5.41秒のクールダウン後、再使用を試行

**再試行メカニズム**（v2.8の新機能）：
- 最大5回まで自動再試行
- 使用後1秒待機してマナ状態を確認
- マナが30%以上または回復傾向なら成功と判定
- 失敗時は0.5秒後に再試行

### 4. フラスコ自動化

**マナフラスコ（2キー）**：
- 4.5-4.8秒間隔で継続的に使用
- Tincture再使用時にタイミングをリセット
- マクロ実行中は常に動作

### 5. スキル自動化

**実行されるスキル**：
- **E, R キー**：1-1.1秒間隔でランダムに実行
- **T キー**：4.1-4.2秒間隔で実行
- **4 キー**（Wine of the Prophet）：動的間隔（上記参照）

### 6. ロード画面検出（ユーザー入力待機式）

**検出方法**：
- 画面下部のGGGロゴ（金色の歯車）を検出
- UI要素（マナ/ヘルスオーブ）の非表示を確認

**動作フロー**：
1. ロード画面検出 → マクロ自動停止
2. ゲーム画面復帰を検出
3. ユーザーの入力（移動、スキル使用など）を待機
4. 入力検出後0.3秒でマクロ自動再開

**検出する入力**：
- 移動キー（WASD、矢印キー）
- マウスクリック（左、右、中）
- スキル使用（Q、E、R、T）
- フラスコ使用（1-5キー）

## 想定される動作フロー

### マップ開始時
1. エリアに入る
2. プレイヤーが移動開始（任意のキー/クリック）
3. マクロが自動的に起動
4. Tincture使用 → マナフラスコ開始 → スキルループ開始

### 戦闘中
1. **通常時**：
   - E, R, Tを自動実行
   - マナフラスコを定期使用
   - Wine of the Prophetを適切な間隔で使用

2. **マナ枯渇時**：
   - Tinctureのバフが切れる
   - マナが0/26まで低下
   - 5.41秒後にTincture再使用
   - マナフラスコのタイミングリセット

### エリア移動時
1. ロード画面を自動検出
2. 全ての自動機能を一時停止
3. 新エリアでプレイヤーの行動を待つ
4. 行動検出後に自動再開

## 必要な環境

- **AutoHotkey**: v2.0以降
- **解像度**: 3440x1440（ウルトラワイド）推奨
- **対応ウィンドウ**: 
  - Path of Exile (PathOfExileSteam.exe)
  - Steam Remote Play (streaming_client.exe)

## インストールと使用方法

1. **AutoHotkey v2.0**以降をインストール
2. すべてのファイルを`PoE-Macro`フォルダに配置
3. `Main.ahk`を実行
4. Path of ExileまたはSteam Remote Playウィンドウをアクティブに
5. **F12**キーでマクロのオン/オフ

## 設定のカスタマイズ

`Config.ahk`ファイルで以下の設定を変更できます：

### キー設定
```autohotkey
global KEY_TINCTURE := "3"      # Tinctureキー
global KEY_MANA_FLASK := "2"    # マナフラスコキー
global KEY_SKILL_E := "E"       # スキルEキー
global KEY_SKILL_R := "R"       # スキルRキー
global KEY_SKILL_T := "T"       # スキルTキー
global KEY_WINE_PROPHET := "4"  # Wine of the Prophetキー
```

### タイミング設定（ミリ秒）
```autohotkey
global TIMING_SKILL_ER := {min: 1000, max: 1100}   # E,Rキーの間隔
global TIMING_SKILL_T := {min: 4100, max: 4200}    # Tキーの間隔
global TIMING_FLASK := {min: 4500, max: 4800}      # フラスコの間隔
global TIMING_MANA_DEPLETED_CD := 5410              # Tinctureクールダウン
```

### その他の設定
```autohotkey
global g_tincture_retry_max := 5    # Tincture再試行回数
global g_debug_mode := false         # デバッグモード
global g_log_enabled := false        # ログ記録
```

## 重要な注意事項

### ゲーム内設定
1. **Always Show Mana Cost**を必ず**OFF**にする
2. リザーブされたマナの表示を**OFF**にする
3. UI設定は変更しない（マナオーブの位置が変わるため）

### 解像度について
- このマクロは**3440x1440**解像度用に最適化されています
- 他の解像度では`Config.ahk`の座標設定の調整が必要です
- `Ctrl+S`で座標を確認しながら調整してください

### パフォーマンスへの影響
- マナ監視：100ms間隔（1秒に10回）
- ロード画面検出：250ms間隔（1秒に4回）
- 通常のプレイへの影響は最小限です

## トラブルシューティング

### マクロが動作しない
1. AutoHotkey v2.0以降がインストールされているか確認
2. ゲームウィンドウがアクティブか確認
3. `F6`でログを確認
4. 管理者権限で実行してみる

### マナ検出が正しくない
1. ゲーム内設定を確認（上記参照）
2. `F11`でマナデバッグ表示を確認
3. 検出ポイントの色を確認
4. UI設定が変更されていないか確認

### Tinctureが再使用されない
1. マナが完全に枯渇（0/26）しているか確認
2. `F7`でTincture状態を確認
3. 再試行回数が上限に達していないか確認

### ロード画面検出が機能しない
1. `F10`で機能が有効か確認
2. 画面下部のGGGロゴが表示されているか確認
3. フルスクリーンモードで実行しているか確認

### スキルが実行されない
1. キーバインドがゲーム内設定と一致しているか確認
2. `F8`でタイマーが動作しているか確認
3. ウィンドウがアクティブか確認

## 既知の制限事項

1. **解像度依存**：3440x1440以外では座標調整が必要
2. **言語依存**：英語版でのみテスト済み
3. **UIスケール**：100%以外のUIスケールには非対応
4. **ウィンドウモード**：フルスクリーンまたはウィンドウフルスクリーン推奨

## 開発者向け情報

### 新機能の追加方法
1. `Features/`ディレクトリに新しいファイルを作成
2. `Main.ahk`でインクルード（依存関係に注意）
3. 必要なグローバル変数を`Config.ahk`に追加
4. ホットキーが必要な場合は`Hotkeys/`に追加

### デバッグ方法
1. `Ctrl+L`でログ記録を有効化
2. `Ctrl+D`でデバッグモードを有効化
3. `logs/`ディレクトリのログファイルを確認
4. `F8`でタイマー状態を確認
5. `F7`で全体の状態を確認

### コード規約
- 関数名：PascalCase（例：`CheckManaRadial`）
- 変数名：snake_case with g_prefix（例：`g_macro_active`）
- 定数：UPPER_CASE（例：`TIMING_MANA_DEPLETED_CD`）
- コメント：日本語可
